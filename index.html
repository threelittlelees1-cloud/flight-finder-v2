GitHub-Powered Deal Watcher

Let GitHub Actions keep scanning flight deals while your site stays simple.

This version uses GitHub Actions to refresh cached fare data from Travelpayouts on a schedule, then publishes the latest deals through a static GitHub Pages site. No always-on server needed.

This is a bargain tracker, not a live booking engine. GitHub updates the data on a timer, so prices may lag behind real checkout screens.

GitHub Setup

The only moving parts
1. Add your Travelpayouts token In GitHub, open Settings → Secrets and variables → Actions and create a repository secret named TRAVELPAYOUTS_API_TOKEN.
2. Turn on GitHub Pages Publish this repo from the main branch root so this single index.html file becomes your live site.
3. Trigger the scan workflow Run the scan-flights workflow manually once, then let the schedule keep it updated.
